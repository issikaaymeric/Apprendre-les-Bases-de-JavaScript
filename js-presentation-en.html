Async/Await (ES8)

async function getData() {
    try {
        const res = await fetch('/api');
        const data = await res.json();
        return data;
    } catch (err) {
        console.error(err);
    }
}

üí° async/await makes asynchronous code look synchronous and easier to read

Why jQuery?

  • Simplified DOM manipulation
  • Easy event handling
  • Built-in animations
  • AJAX made simple
  • Cross-browser compatibility

‚ö†Ô∏è Common Beginner Mistakes

  • Using = instead of === for comparison
  • Forgetting to convert inputs with Number() or parseInt()
  • Arrays start at index 0, not 1
  • Running code before DOM is loaded
  • Not handling null or undefined
  • Forgetting parentheses when calling functions: myFunc vs myFunc()
  • Confusing let/const scope with var
  • Not using async/await properly with Promises

üí° Place <script> at the end of <body> or use defer attribute
